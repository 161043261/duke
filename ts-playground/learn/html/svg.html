SVG, Scalable Vector Graphic

    优点:
  1. 图形尺寸较小, 但高度可缩放, 不会像素化
  2. 文件大小较小, 只需要存储算法, 不需要存储每个像素的信息
  3. 内联 svg 可以减少 http 请求, 减少加载时间
  4. 可以为 svg 标签分配 class 和 id, 并使用 css
  5. 可以将 svg 标签包裹在 a 标签中, 成为超链接
    6. 缺点:
  6. 只适用于 svg 单次使用的情况, 多次使用会导资源密集型维护 (resource-intensive maintance)
  7. 额外的 svg 代码会增大 HTML 文件的大小
  8. 浏览器可以缓存普通图像, 但不能缓存内联 svg

    位图和矢量图

  • 位图使用像素网格定义, 位图精确包含每个像素的位置和颜色
    常见的 web 位图格式包括: bitmap(.bmp), png(.png), jpeg(.jpg), gif(.gif)
  • 矢量图基于 XML, 使用算法定义, 矢量图包含图形和路径的定义
    使用算法计算图形, 根据放大的倍数调整算法中的值
